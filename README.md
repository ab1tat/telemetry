# 📊 aBitat-Telemetry-API — Central Processing Hub

The Telemetry API acts as the centralized brain for the **aBitat** platform, receiving, processing, and serving all telemetry data generated by the **Argus** (inTropic) engine.

## 🛰️ Role
- **Data Aggregator:** Receives high-volume entropy and risk data from intelligence agents.
- **Persistence:** Efficiently stores telemetry metadata for historical analysis and trend detection.
- **Dashboard Bridge:** Supplies the `Papyros` frontend with real-time analytics through RESTful endpoints.

## ⚙️ Architecture (ASP.NET Core 8)
- **Controllers:** Exposes `/api/telemetry` for agents and dashboard consumption.
- **Repository Pattern:** Decoupled data access for enhanced maintainability.
- **Response Models:** Standardized JSON responses for seamless frontend integration.

## 🛠️ Usage

### API Endpoints
- `GET /api/telemetry?count=50`: Lists the latest telemetry logs.
- `GET /api/telemetry/{id}`: Retrieves detailed information for a specific log entry.
- `POST /api/telemetry`: Endpoint for agent data submission (Argus).

### Setup
```bash
dotnet build
dotnet run --urls http://localhost:5073
```
